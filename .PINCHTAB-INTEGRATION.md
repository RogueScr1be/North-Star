# Pinchtab + North Star Integration Guide

## Overview

**Pinchtab** is a lightweight (12MB), token-efficient browser automation tool built for AI agents. It replaces expensive screenshot-based verification with text extraction (5-13x token savings) and accessibility-first element selection.

## Architecture

```
North Star Development
├── Frontend (React + Vite) → localhost:3000
├── Backend (Express) → localhost:3001
├── Pinchtab Server → localhost:9867 (NEW)
│   ├── Headless Chrome instances (managed by Pinchtab)
│   └── HTTP API for automation
└── Testing/Verification Scripts
    └── pinchtab-test.sh (reference implementation)
```

## Installation

Pinchtab is already built and ready to use:

## Quick Start

### 1. Start Pinchtab Server (Terminal 1)

```bash

```

Expected output:

```bash

```

### 3. Run Integration Test (Terminal 3)


- ✓ Launch a headless Chrome instance
- ✓ Navigate to constellation page
- ✓ Extract page structure (text-based, not screenshot)
- ✓ Show token efficiency comparison
- ✓ Demonstrate element selection

## API Reference

### Basic Commands (CLI)

```bash
# Navigate to URL
pinchtab nav https://example.com

# Get page structure (interactive elements only)
pinchtab snap -i

# Get page structure with coordinates
pinchtab snap -i -c

# Extract all text
pinchtab text

# Click element by ref
pinchtab click e5

# Fill input
pinchtab fill e3 "value"

# Press key
pinchtab press e7 Enter

# Type text
pinchtab type e3 "hello"
```

### HTTP API (Direct)

```bash
# List running instances
curl http://localhost:9867/instances

# Launch instance
curl -X POST http://localhost:9867/instances/launch \
  -H "Content-Type: application/json" \
  -d '{"name":"test","mode":"headless"}'

# Open tab in instance
curl -X POST http://localhost:9867/instances/{INST_ID}/tabs/open \
  -H "Content-Type: application/json" \
  -d '{"url":"http://localhost:3000/constellation"}'

# Get snapshot
curl "http://localhost:9867/tabs/{TAB_ID}/snapshot?filter=interactive"

# Extract text
curl "http://localhost:9867/tabs/{TAB_ID}/text"

# Click element
curl -X POST http://localhost:9867/tabs/{TAB_ID}/action \
  -H "Content-Type: application/json" \
  -d '{"kind":"click","ref":"e5"}'
```

## Advanced Usage

### Multi-Instance Testing (Parallel)

```bash
# Instance 1: Test search
INST1=$(curl -s -X POST http://localhost:9867/instances/launch \
  -H "Content-Type: application/json" \
  -d '{"name":"test1","mode":"headless"}' | jq -r '.id')

# Instance 2: Test canvas interaction
INST2=$(curl -s -X POST http://localhost:9867/instances/launch \
  -H "Content-Type: application/json" \
  -d '{"name":"test2","mode":"headless"}' | jq -r '.id')

# Both instances run independently, isolated
```

### Headed Mode (Visible Chrome)

For debugging, run in headed mode to see what Pinchtab is doing:

```bash
curl -X POST http://localhost:9867/instances/launch \
  -H "Content-Type: application/json" \
  -d '{"name":"debug","mode":"headed"}'
```

A Chrome window will open showing the automation in real-time.

### Profile Persistence

Profiles survive Pinchtab restarts:

```bash
# First run - logs in
pinchtab nav https://example.com
pinchtab fill e1 "user@example.com"
pinchtab press e2 Enter

# Next run - already logged in (profile remembered)
pinchtab nav https://example.com
# Cookies + localStorage preserved
```

## Token Efficiency Breakdown

### Screenshot Approach (Old)
```
1. Take screenshot → JPEG/PNG image
2. Send to Claude → ~10,000 tokens
3. Claude analyzes pixels
4. Repeat for each verification
```

### Pinchtab Approach (New)
```
1. Get page snapshot (text) → JSON
2. Send to Claude → ~800 tokens
3. Claude analyzes structure + text
4. Repeat for each verification
```

**Savings example:**
- Old approach: 10 verifications × 10,000 tokens = 100,000 tokens
- New approach: 10 verifications × 800 tokens = 8,000 tokens
- **Result: 92,000 tokens saved (12x reduction)**

## Troubleshooting

### Pinchtab Server Won't Start

```bash
# Check if port 9867 is in use
lsof -i :9867

# Kill existing process
kill -9 $(lsof -t -i:9867)

# Try again
~/bin/pinchtab
```

### Health Check

```bash
# Verify server is responding
curl http://localhost:9867/health
# Should return: {"status":"ok"}
```

### Element Refs Not Working

Element refs (e.g., `e5`) are assigned by Pinchtab based on page structure:

```bash
# Always get fresh snapshot before clicking
pinchtab snap -i -c

# Use the ref from current snapshot (not cached)
pinchtab click e5  # This ref is only valid for current snapshot
```

### No Elements Found

```bash
# Snapshot might be showing non-interactive elements
# Use filter to show only clickable elements
pinchtab snap -i  # interactive only
pinchtab snap -i -c  # interactive with coordinates
```

## Integration with North Star Testing

### Example: Automated Constellation Verification

```bash
#!/bin/bash
# Test constellation canvas render + interaction

set -e

PINCHTAB_API="http://localhost:9867"
NS_URL="http://localhost:3000/constellation"

# Launch instance
INST=$(curl -s -X POST $PINCHTAB_API/instances/launch \
  -d '{"name":"test","mode":"headless"}' -H "Content-Type: application/json" | jq -r '.id')

# Open tab
TAB=$(curl -s -X POST $PINCHTAB_API/instances/$INST/tabs/open \
  -d "{\"url\":\"$NS_URL\"}" -H "Content-Type: application/json" | jq -r '.tabId')

# Wait for render
sleep 2

# Verify projects visible
TEXT=$(curl -s $PINCHTAB_API/tabs/$TAB/text)
echo "$TEXT" | grep -q "North Star" && echo "✓ Projects visible"

# Verify search works
SNAP=$(curl -s "$PINCHTAB_API/tabs/$TAB/snapshot?filter=interactive")
SEARCH_INPUT=$(echo "$SNAP" | jq -r '.elements[] | select(.label | contains("Search")) | .ref' | head -1)

if [ ! -z "$SEARCH_INPUT" ]; then
    curl -s -X POST $PINCHTAB_API/tabs/$TAB/action \
      -d "{\"kind\":\"fill\",\"ref\":\"$SEARCH_INPUT\",\"text\":\"decision\"}" \
      -H "Content-Type: application/json"
    sleep 1
    echo "✓ Search works"
fi

# Cleanup
curl -s -X DELETE $PINCHTAB_API/instances/$INST

echo "✓ All tests passed"
```

```bash

pinchtab snap -i -c

# Find search input ref
SEARCH_INPUT=$(pinchtab snap -i | jq -r '.elements[] | select(.label | contains("Search")) | .ref' | head -1)

# Fill and verify
pinchtab fill $SEARCH_INPUT "decision"

# Extract text for verification
pinchtab text | head -30
```

**vs old screenshot approach:**
- Old: Take screenshot, upload, claude analyzes pixels
- New: Get text snapshot, copy/paste into terminal, instant verification
